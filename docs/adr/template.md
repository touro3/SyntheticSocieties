# NNNN. Short imperative title

- Status: proposed
- Date: YYYY-MM-DD
- Deciders: who signed off
- Tags: relevant, areas

## Context

What forces this decision? What constraints, prior work, or pain points
make it necessary? Quote concrete numbers, file paths, or incidents
where you can.

## Decision

The chosen approach, stated as a single declarative sentence. Then the
specifics: which file does what, which module owns which boundary.

## Consequences

**Positive** — what this buys us.

**Negative** — what we give up; what gets harder.

**Mitigation** — how we plan to handle the negatives, or what we
explicitly accept.

## Alternatives rejected

Briefly: each option we seriously considered and why it lost.
