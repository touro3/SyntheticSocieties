# Reproducing the BGF Paper

This document tells an external reviewer how to regenerate every table and figure cited in `docs/paper.md` from a clean checkout. The single-command path is `scripts/reproduce_paper.sh`; this file documents what each stage produces, what is and isn't regenerated, and the GPU split.

## One-command reproduction

```bash
git clone <repo>
cd SyntheticSocieties
bash scripts/reproduce_paper.sh          # fast sanity (CPU, ~5 min)
bash scripts/reproduce_paper.sh --full   # full CPU baseline (~30 min)
bash scripts/reproduce_paper.sh --figures # regenerate figures only
```

The script bootstraps `venv/`, installs `requirements.txt`, runs `pytest tests/`, executes a baseline simulation pipeline at the chosen scale, regenerates trust-gradient and cross-cultural CPU baselines, regenerates figures, emits the BRM Proposition 3 certificate (§6.1), extracts §8.1 cooperation-graph modularity (Table 8 H4), re-aggregates the §8.1 experiment matrix, regenerates the §6.0 hypothesis forest plot, and checks Zenodo metadata. All on CPU.

## What `reproduce_paper.sh` does **not** regenerate

The LLM-policy experiments (Conditions A, B, C; §6.1 pilot, §8.1 N=100 extension, in-flight N=500 extension, cross-model panel, memory ablation) require GPU and live model weights, take 6–80 GPU-hours per cell, and are **excluded from CI and from the one-command reproduction path by design**. They are documented as separate launchers:

```bash
# §8.1 N=100 confirmatory extension (~7 h per cell × 20 cells; needs P100-class GPU)
python scripts/run_experiment_matrix.py --include-llm \
    --conditions A B --seeds 1..10 --rounds 30 --agents 100 --skip-existing

# §6.0a single-seed N=50 patched-code re-execution (~3 h)
# (replicates the cmp_llm_s42_condA/condB cells)

# §8.5 H8 memory ablation (LLM policy, ~6–8 GPU-h)
bash scripts/run_memory_ablation_llm.sh

# §6.6 cross-model panel (Mistral + Qwen + GPT-4o-mini; needs OPENAI_API_KEY for the third)
python scripts/run_cross_model_comparison.py

# §8.3 cross-cultural LLM-scale (~3–4 GPU-h at N=20 × 10 seeds × 6 clusters)
bash scripts/pipeline_cross_cultural.sh --include-llm --n-seeds 10
```

After any of those runs lands new `experiments/mx_*` directories on disk, re-run `bash scripts/reproduce_paper.sh` and the stages above (8, 9a, 9b, 9c) will fold the new cells into the regenerated CSVs and the forest-plot PNG automatically.

## Continuous integration

`.github/workflows/ci.yml` runs the test suite (`pytest tests/ --cov-fail-under=70`), ruff lint+format check, mypy (non-blocking), and Vue frontend build on every push/PR to `main`. It additionally emits a fresh BRM Proposition 3 certificate before the tests so `tests/test_brm_certificate_emission.py` guards the JSON shape contract on every commit. The CI deliberately does **not** regenerate the paper tables (the full CPU baseline + figure regen would blow the runner budget); paper-table regeneration is the local-machine job of `scripts/reproduce_paper.sh --full`. Hugging Face Space deployment is triggered on every merge to `main` (Settings → Secrets: `HF_TOKEN`, `HF_SPACE_ID`).

## Forensic-trail-only artefacts

The historical forensic record — withdrawn pilot `B_RLHF` values, the L-1/L-2/C-1 infrastructure bugs that motivated the patched-code re-execution, and the superseded Qwen2.5-7B / GPT-4o-mini cross-model rows — is preserved in `docs/appendix_audit_trail.md`. The two forensic JSON files (`analysis/reports/b_rlhf_pilot_forensic.json` and `analysis/reports/cross_model_b_rlhf_forensic.json`) are checked in as static evidence and are not regenerated by `reproduce_paper.sh`; they are point-in-time records of what was withdrawn and why, and re-deriving them requires the pre-patch code, which is preserved only in git history (commits prior to `c73554e`).
